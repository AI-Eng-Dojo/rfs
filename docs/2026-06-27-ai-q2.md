# AI Q2

## Summary

AI Q2 is a cloud-hosted personal AI agent for people who want work to feel more enjoyable and less isolating. It combines long-term memory, a strong original avatar personality, text chat, and paid task execution.

The first version is intentionally narrow: users talk with the agent in text chat, ask for work or personal advice, and can request concrete tasks. For task requests, the service estimates the required token usage with roughly a 200% buffer, shows the price before execution, and sends the result to the user's registered email address.

## Problem

Many AI assistants are useful but transactional. They answer a prompt, then the user has to rebuild context, tone, preferences, and working style in the next session. That makes them feel less like a long-term partner and more like a command line with natural language.

At the same time, many people want work to feel more fun. The first wedge is not "everyone" in practice, but individual knowledge workers, founders, freelancers, creators, and small-team operators who would benefit from a persistent AI work companion with a memorable personality.

The pain is strongest when users want:

- a familiar agent that remembers long-running context
- a more enjoyable way to ask for work help
- a single place for both casual consultation and task execution
- asynchronous delivery when a task takes longer than a chat response

## Why Now

The opportunity depends on several changes becoming practical at the same time: better general-purpose language models, stronger agentic task execution, cheaper cloud inference options, more user acceptance of AI chat products, and growing expectations that AI tools should remember context across sessions.

The timing question is whether users now want a personal agent that is not only capable, but also distinct enough in personality that they return to it weekly.

## First Product

The MVP should focus on text chat and email delivery.

- Persistent text chat with long-term user memory
- A strong, original avatar personality
- Explicit memory controls so users can view, edit, or delete remembered facts
- Monthly subscription around 500 JPY
- Usage-based chat or task charges
- Task request flow with a rough token estimate, 200% buffer, and pre-execution price confirmation
- Email delivery for completed task results

Voice chat and LINE delivery are future options, not MVP requirements.

The first tasks should stay bounded: drafting, rewriting, research summaries, planning, inbox-style triage, document outlines, and other work that can be delivered as text. The product should avoid external actions, purchases, account operations, or high-stakes advice until permissioning, audit logs, and liability boundaries are clearer.

## Potential Users

Start with users who already like AI tools but want the experience to feel more personal and habit-forming:

- solo founders who want a persistent thinking partner
- freelancers who want lightweight task help and encouragement
- creators who want an always-available planning and drafting companion
- individual knowledge workers who want work consultation without setting up a full agent stack

The initial positioning should be "make work feel more fun with a capable personal agent", not a generic assistant for everyone.

## Validation

The first validation signal is whether users consult AI Q2 at least once per week after onboarding.

Stronger signals include:

- users voluntarily add or correct memories
- users pay beyond the 500 JPY monthly subscription
- users accept quoted task prices after seeing the estimate
- users return to the same agent for both personal consultation and work execution
- users say the personality is a reason they keep using the product

## Open Questions

- Can the product avoid becoming a novelty avatar chat app with weak retention?
- Is a 500 JPY monthly subscription high enough to cover memory storage, inference, task execution, and support?
- Does a 200% token buffer protect margins without making quoted task prices feel too expensive?
- How should the product handle privacy, sensitive personal memories, deletion requests, and data portability?
- What level of task autonomy is safe before the product needs audit logs, approvals, and stronger permission controls?
- How does the product differentiate from general-purpose chatbots, companion AI apps, and agent execution tools?
- Could a strong personality create trust, dependency, or safety issues for vulnerable users?
- Which tasks are bounded enough for reliable delivery by email without creating refund or support risk?
